<Update label="July 13, 2026" description="Week of July 6 – July 13">

## New features

- **Redesigned portfolio page.** Your portfolio has a refreshed layout with an updated pie chart, clearer holdings breakdown, and improved navigation between positions. Track your Slices, funds, and WLTH at a glance.
- **Global whitelist for "Buy now".** Whitelisted investors can now complete the buy-now flow across the app, including the [/gift](/investment/slices/gifting-slices) flow, without hitting access checks on individual funds or companies.
- **Login deep link.** Sharing a link that includes a login parameter now opens the Privy login modal automatically, making it easier to onboard new users from campaigns and referrals.

## Updates

- **Referral code handling improvements.** The referral experience in login and referral screens is more reliable, including support for WHALES referral codes end-to-end. See the [Referrals](/referrals) page.
- **In-app updater (mobile).** Mobile now uses a native update prompt and supports forced updates so you're always on the latest version. Android is live at v1.2.8.
- **Coming soon fund cards.** Fund cards in "Upcoming" and "Coming soon" states now display correctly, with the register form hidden until a fund opens. Learn more about [WLTH Funds](/investment/exclusive-access/wlth-funds).
- **PWA icons.** Refreshed home-screen icons for a cleaner install experience.

## Bug fixes

- **Whitelist funds ordering.** Fixed an issue where whitelisted funds displayed in the wrong order.
- **Upcoming and coming soon statuses.** Corrected status labels and details pages for funds in pre-launch states.
- **Insufficient ERC-20 balance (mobile).** Transfers now surface a clear error before submission when your ERC-20 balance is too low, instead of failing on-chain.
- **Investment amount display (mobile).** Fixed an error that could appear when displaying large investment amounts on the results screen.
- **WLTH growth card.** Corrected the percentage display on the WLTH growth card.

</Update>
